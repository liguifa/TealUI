    ☍事件
    util/events
  • 复制 import "util/events"
  • 复制 require("util/events")
  • 复制 "util/events"
    •

    提供事件管理对象。

    事件是一种设计模式。 程序可以通过触发事件的方式通知外部。 对此通知感兴趣的对象可以订阅事件并执行相应的回调函数。 通过事件的设计模式可以将触发事件和实际处理事件的逻辑完全分开，提升代码的可重用性。

    另参考

    Node.js: Event

    ☍API

    ☍⅏EventEmitter 类

    表示一个事件分发器。

    方法 描述
    ⅏ ⮞ on(eventName, eventListener)(eventName:string, eventListener:Function):this

    添加一个事件监听器。

    参数 类型 描述 默认值
    eventName* string

    要添加的事件名。

    		 —
    eventListener* Function

    要添加的事件监听器。

    		 —

    返回值

    类型：this

    示例

    ❐new EventEmitter().on("click", function (e) {  });

    添加一个事件监听器。
    ⅏ ⮞ off(...)(eventName?:string, eventListener?:Function):this

    删除指定或所有的事件监听器。

    参数 类型 描述 默认值
    eventName string | undefined

    要删除的事件名。如果不传递此参数，则删除所有事件监听器。

    		 —
    eventListener Function | undefined

    要删除的事件监听器。如果不传递此参数，则删除指定事件的所有监听器。如果同一个事件被多次添加，则只删除第一个。

    		 —

    返回值

    类型：this

    示例

    ❐var fn =  function () {}; // 必须保存函数的引用才能删除。
var ee = new EventEmitter();
ee.on("click", fn); // 绑定一个 click 事件。
ee.off("click", fn); // 删除一个 click 事件。

    删除指定或所有的事件监听器。
    ⅏ ⮞ emit(eventName, ...)(eventName:string, ...eventArgs:any[]):boolean

    触发一个事件。

    参数 类型 描述 默认值
    eventName* string

    要触发的事件名。

    		 —
    eventArgs any[]

    传递给监听器的所有参数。

    		 —

    返回值

    类型：boolean

    如果事件被阻止则返回 false，否则返回 true。

    示例

    ❐var ee = new EventEmitter(); // 创建一个实例。
ee.on("click", function (e) { console.log("haha"); }); // 绑定一个 click 事件。
ee.emit("click"); // 手动触发 click。

    触发一个事件。
