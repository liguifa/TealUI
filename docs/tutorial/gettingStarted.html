    ☍概述

    TealUI 的目的是打造一个完整的前端解决方案，它提供了前端项目所需要的大部分功能：

    • UI 组件
    • CSS 设计框架
    • 工具函数库
    • 开发规范和构建工具

    尽量功能众多，但每个功能都被拆成独立的小组件，项目里可单独引用。

    ☍Hello World

    相比社区内的组件，TealUI 的组件用法统一、体积更小、性能更优。 只需掌握几个组件的用法，其它组件的用法都大同小异，学起来非常容易。

    现在以拖动组件为例，演示如何快速使用一个组件。

    1. 新建一个名为 hello.html 的文件，内容如下：
    ❐<!DOCTYPE html>
<html>
<head>
    <title>我的第一个页面</title>
    <script src="http://tealui.com/components/require.js"></script>
</head>
<body>
    
</body>
</html>

    这个页面引入了一个名为 require.js 的文件，这个文件定义了全局的 require 函数用来导入组件。

    假如您下载了组件库，可以改成本地的 require.js 路径。

    1. 导入 ux/drag 组件实现拖动效果。
    ❐<!DOCTYPE html>
<html>
<head>
    <title>我的第一个页面</title>
    <script src="http://tealui.com/components/require.js"></script>
</head>
<body>
    <span id="box">Hello world!</span>
    <script>
        require(["ux/drag"], function (drag) {
            var draggable = drag.default;

            draggable(document.getElementById("box"));
        });
    </script>
</body>
</html>

    在这个示例中，使用了 require 导入了 ux/drag 模块，通过这个模块导出的 default 函数接口实现了拖动效果。

    所有组件都可以通过相对于 require.js 的相对地址或者绝对地址导入，导入一个组件时组件会自动导入其内部的依赖组件。

    组件本质上就是一个符合 AMD 模块规范的 JavaScript 文件或一个 CSS 文件。 导入 JavaScript 组件后可直接使用其导出的 API 接口。 导入 CSS 组件后可以直接使用其内部定义的所有 CSS 类。

    点击页面顶部的组件可以查看 TealUI 提供的所有组件列表，每个组件文档顶部会显示该组件的引入路径，底部会列出该组件导出的所有 API 接口。

    1. 如果使用了预构建工具，则可以使用 import 语法简化组件导入。
    ❐<!DOCTYPE html>
<html>
<head>
    <title>我的第一个页面</title>
    <script src="http://tealui.com/components/require.js"></script>
</head>
<body>
    <span id="box">Hello world!</span>
    <script>
        import draggable from "ux/drag";
        
        draggable(document.getElementById("box"));
    </script>
</body>
</html>

    import 语法并不是使用组件的必要条件，详细说明参考 MDN。

    ☍项目脚手架

    现在你已经掌握了如何使用组件的基本方法。

    在实际项目中，一般是通过预构建的方式来导入组件的，相比通过 require.js 导入组件的方式，它有更多的优势：

    1. 可以使用更简短的 import 语法。
    2. 合并导入的组件，减少请求数，提升页面性能。
    3. 支持在导入时压缩优化代码。

    TealUI 所有组件均使用 TypeScript + Sass 编写，并且遵循 commonjs 规范。 因此可以使用任何一个支持 commonjs 规范的构建工具导入组件源码。

    TealUI 默认发布后的代码为标准 JavaScript 和 CSS 文件，其遵循 AMD 规范。 因此也可以使用任何一个支持 AMD 规范的构建工具直接导入发布后的组件。

    下文将介绍如何配置 Webpack 和 Digo 导入组件。

    ☍配置 Webpack

    1. 新建 package.json，内容如下：
    ❐{
    "dependencies": {
        "tealui": "latest"
    },
    "devDependencies": {
        "webpack": "latest",
        "webpack-dev-server": "latest"
    },
    "scripts": {
        "build": "webpack"
    }
}
    1. 安装依赖包
    ❐npm install

    这个命令会安装 package.json 中指定的依赖包。

    1. 新建 webpack.config.js，内容如下：
    ❐module.exports = {
    entry: './src/index.js',
    output: {
        filename: 'bundle.js',
        path: __dirname + '/dist'
    }
};
    1. 新建 src/index.js，内容如下：
    ❐import MessageBox from "tealui/ui/messageBox";

MessageBox.alert("Hello World!");
    1. 新建 dist/index.html，内容如下：
    ❐<!DOCTYPE html>
<html>
<head>
    <title>我的第二个页面</title>
    <script src="bundle.js"></script>
</head>
<body>

</body>
</html>
    1. 执行以下命令构建：
    ❐webpack

    执行后项目会编译 src/index.js 文件生成 dist/bundle.js。

    1. 在浏览器打开 dist/index.html。 可以看到消息框组件已成功加载并显示了一个弹窗。

    1. 直接导入编译后的代码

    1. 导入源码并使用对应的 webpack loader 即时编译。
